# Issue 10 (v1.1): Settings panel + GM_setValue persistence

> Milestone: v1.1
> Triage label: `ready-for-agent`
> Blocked by: None

## What to build

Add a settings panel so power users can disable per-site rules and override dark-mode auto behavior. Settings persist via `GM_setValue` with `reader-mode:` namespace prefix.

This unlocks per-site opt-out (previously blocked: v1 was 0-config). When a site is disabled, its `apply()` is replaced with a no-op.

## Acceptance criteria

- [ ] `src/utils/ui.js` exports `openSettingsPanel()` which renders a floating panel
- [ ] Tampermonkey menu item "reader-mode 设置" opens the panel (via `GM_registerMenuCommand`)
- [ ] Panel shows: per-site checkbox list (5 sites) + dark mode toggle (auto / on / off)
- [ ] State persisted via `GM_setValue('reader-mode:disabled-sites', [...])` and `GM_setValue('reader-mode:dark-mode', 'auto' | 'on' | 'off')`
- [ ] On script load, disabled sites have their `apply()` short-circuited
- [ ] Dark mode toggle overrides `@media (prefers-color-scheme: dark)` when set to "on" or "off"
- [ ] Panel UI does not break site layout (z-index managed, escape key closes)
- [ ] Manual smoke test: disable CSDN, reload CSDN page, verify nothing happens; re-enable, verify it works
