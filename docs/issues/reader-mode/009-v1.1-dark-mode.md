> Milestone: v1.1

## What to build

Add dark mode to reader-mode. v1.1 strategy: **auto-follow `prefers-color-scheme: dark`** — no UI toggle needed (deferred to Issue #10 when settings panel arrives). When the user's OS is in dark mode, reader-mode inverts the article body's color scheme while preserving the site's chrome (or fully restyles for max readability — designer's choice, document decision in code).

## Acceptance criteria

- [ ] Dark mode CSS block injected when `@media (prefers-color-scheme: dark)` matches
- [ ] Article body has dark background + light text + reduced link color saturation
- [ ] Code blocks (where supported) have a dark-friendly theme override
- [ ] Verified on at least CSDN + 知乎 (other sites pick up automatically via `applyLayoutPolish`)
- [ ] Toggle on/off by switching OS theme; reader-mode follows without reload
- [ ] No flash of light theme on page load (style injected at `document-start` or shadowed to prevent FOUC)
